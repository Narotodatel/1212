# ОТЧЁТ О ВЫПОЛНЕНИИ ЗАДАНИЯ 1

## Создание программного продукта

Во время выполнения первого задания было разработано приложение для работы с электронными книгами под названием «BookPlatform». Основная цель проекта — создание платформы, на которой пользователи могут просматривать книги, оставлять отзывы, вести собственные списки чтения и публиковать произведения.

В процессе разработки была спроектирована и реализована база данных в Microsoft SQL Server. Структура базы данных включает таблицы пользователей, ролей, книг, жанров, отзывов, списков чтения, жалоб и заявок. Все таблицы связаны между собой с помощью внешних ключей, а структура приведена к третьей нормальной форме.

Для взаимодействия приложения с базой данных использовалась технология Entity Framework Database First. Был создан вспомогательный класс Core, обеспечивающий доступ к контексту базы данных из разных частей приложения.

### Листинг 1 — Класс подключения к базе данных

```csharp
internal class Core
{
    public static BookPlatformEntities Context =
        new BookPlatformEntities();
}
```

В приложении реализована система авторизации и регистрации пользователей. После успешного входа выполняется переход в главное окно программы, где пользователю становятся доступны разделы в зависимости от его роли.

### Листинг 2 — Метод авторизации

```csharp
private void LoginBtn_Click(object sender, RoutedEventArgs e)
{
    var user = Core.Context.Users.FirstOrDefault(x =>
        x.Login == LoginTB.Text &&
        x.PasswordHash == PasswordPB.Password);

    if (user != null)
    {
        CurrentUser.User = user;

        MainWindow window = new MainWindow();
        window.Show();

        Close();
    }
    else
    {
        MessageBox.Show("Неверный логин или пароль");
    }
}
```

Также была реализована регистрация новых пользователей с автоматическим назначением роли «Читатель».

### Листинг 3 — Метод регистрации

```csharp
private void RegisterBtn_Click(object sender, RoutedEventArgs e)
{
    int readerRoleId = Core.Context.Roles
        .First(x => x.RoleName == "Читатель")
        .RoleID;

    Users user = new Users()
    {
        Login = LoginTB.Text,
        PasswordHash = PasswordPB.Password,
        Email = EmailTB.Text,
        DisplayName = NameTB.Text,
        RoleID = readerRoleId,
        IsFrozen = false,
        CreatedAt = DateTime.Now
    };

    Core.Context.Users.Add(user);

    Core.Context.SaveChanges();
}
```

Дополнительно были реализованы:

* просмотр каталога книг;
* поиск и сортировка;
* система отзывов;
* жалобы на книги и отзывы;
* списки чтения;
* функционал администратора и автора.

## Улучшение и переработка кода

На втором этапе работы был выполнен рефакторинг ранее написанного кода. Основная задача заключалась в повышении читаемости проекта, уменьшении повторений и упрощении дальнейшей поддержки приложения.

В процессе переработки были выполнены следующие изменения:

1. Повторяющиеся фрагменты кода были вынесены в отдельные методы.

2. Улучшена навигация между страницами приложения.

3. Добавлены дополнительные проверки корректности пользовательских данных.

4. Реализована обработка возможных ошибок при работе с изображениями книг.

5. Улучшена структура страниц и распределение логики между XAML и code-behind.

В качестве примера можно привести изменение загрузки изображений книг.

### Листинг 4 — Загрузка изображения до переработки

```csharp
image.Source = new BitmapImage(
    new Uri(book.CoverImagePath));
```

После переработки был добавлен полный путь к изображению и обработка исключений.

### Листинг 5 — Загрузка изображения после переработки

```csharp
try
{
    image.Source = new BitmapImage(
```

# ОТЧЁТ О ВЫПОЛНЕНИИ ЗАДАНИЯ 3

## Разработка интерфейса WPF-приложения

На заключительном этапе был создан графический интерфейс приложения с использованием технологии WPF.

Интерфейс приложения выполнен в тёмной цветовой схеме и состоит из нескольких страниц.

### 1) Главное окно приложения

Главное окно содержит боковую панель навигации с кнопками перехода между разделами:

* каталог книг;
* списки чтения;
* профиль;
* страница автора;
* панель администратора.

Для навигации использовался элемент Frame.

---

### 2) Каталог книг

На странице каталога реализован вывод книг в виде карточек. Для каждой книги отображаются:

* обложка;
* название;
* автор;
* средняя оценка.

Также реализованы:

* поиск;
* сортировка;
* фильтрация.

---

### 3) Страница книги

На странице книги отображается полная информация:

* название;
* описание;
* текст книги;
* жанры;
* отзывы пользователей.

Пользователь может:

* оставить отзыв;
* поставить оценку;
* пожаловаться на книгу или отзыв;
* добавить книгу в список чтения.

---

### 4) Списки чтения

Была реализована система пользовательских списков:

* «В планах»;
* «Читаю»;
* «Прочитано»;
* «Заброшено».

Пользователь может перемещать книги между разделами.

---

### 5) Панель администратора

Страница администратора предназначена для управления системой. Реализованы:

* просмотр жалоб;
* обработка заявок;
* управление ролями пользователей;
* заморозка книг и отзывов;
* изменение данных пользователей.

---

### 6) Страница автора

Для пользователей с ролью автора создан отдельный раздел, в котором доступны:

* добавление новых книг;
* редактирование существующих;
* просмотр замороженных книг;
* отправка заявок на разморозку.

---

### 7) Профиль пользователя

На странице профиля отображается информация о текущем пользователе:

* логин;
* имя;
* электронная почта;
* роль.

Также реализован просмотр собственных отзывов и возможность отправки заявки на получение роли автора.

---

В результате выполнения практической работы было разработано полноценное WPF-приложение для работы с электронными книгами, использующее базу данных SQL Server и Entity Framework.
